- `.spec/requirements.md` - Functional and non-functional requirements
- `.spec/architecture.md` - Technical architecture and patterns
- `.spec/implementation-plan.md` - Implementation phases and tasks

## Instructions

1. **Understand the task**: Read the user's request carefully
2. **Check specifications**: Verify the feature is in requirements.md
3. **Follow architecture**: Use patterns from architecture.md
4. **Implement**: Write clean, typed TypeScript code
5. **Test**: Ensure the feature works correctly

## Code Standards

- Use Angular 19 standalone components
- Use Signals for state management
- Use Carbon Design System components
- Follow strict TypeScript (no `any`)
- Use `ChangeDetectionStrategy.OnPush`
- Add JSDoc comments for public APIs

## File Naming

- Components: `feature-name.component.ts`
- Services: `feature-name.service.ts`
- Stores: `feature-name.store.ts`
- Models: `feature-name.model.ts`

Now implement the requested feature following these guidelines.
